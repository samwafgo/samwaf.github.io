# IP数据库管理

## 功能简介

IP数据库管理用于配置和管理 SamWaf 的 IP 归属地查询能力。SamWaf 会根据上传的离线 IP 数据库，对访问来源 IP 解析出国家、省份、城市、运营商等归属地信息。本页面支持分别为 IPv4 与 IPv6 配置数据来源，上传数据库文件，以及在线测试某个 IP 的查询结果。

支持的数据来源：

- **ip2region**：xdb 格式离线库，可用于 IPv4 与 IPv6，并支持多种字段格式。
- **GeoLite2**：mmdb 格式离线库，可用于 IPv6。
- **IPDB**：ipdb 格式离线库，同时覆盖 IPv4 和 IPv6 查询。

::: tip 开箱即用，无需先上传
SamWaf 已内置两份 IP 数据库，随程序一起发布：**IPv4 的 ip2region**（老版本格式）与 **IPv6 的 GeoLite2**。全新安装后无需上传任何文件，IP 归属地查询即可直接使用；这两个来源在页面上会标记为「内置」。

上传同类型的文件会覆盖内置数据。其余来源没有内置数据，必须先上传才能使用：**IPv6 的 ip2region**（需上传 `ip2region_v6.xdb`）和 **IPDB**（需上传 `.ipdb` 文件）。
:::

## 操作步骤

### 一、测试 IP 查询

页面顶部固定有 IP 查询测试区域，可随时验证当前数据库的查询效果：

1. 在 **测试IP地址** 输入框填入要查询的 IP（支持 IPv4 和 IPv6），也可点击输入框旁帮助图标中的示例 IP（如 Google IPv4 `8.8.8.8`、IPv6 `2001:4860:4860::8888`）自动填入。
2. 点击 **查询**。
3. 查询结果会显示 IP 类型、实际使用的数据源（ip2region 还会显示使用的格式），以及国家、省份、城市、运营商、区域、区县等信息。

### 二、上传数据库文件

切换到 **上传数据库** 标签页：

1. 在 **上传IPv4数据库** / **上传IPv6数据库** 处上传 `.xdb` 或 `.mmdb` 文件；或在 **上传IPDB文件** 处上传 `.ipdb` 文件（同时覆盖 IPv4 和 IPv6 查询）。
2. 上传成功后，点击 **重新加载数据库**，使新上传的文件立即生效。
3. 本页面下方提供 ip2region 的字段格式说明表，可对照所用 xdb 文件版本选择正确格式。

### 三、配置数据来源

切换到 **配置与状态** 标签页：

1. 在 **IPv4数据库配置** 中选择数据来源（ip2region / IPDB）。若来源为 ip2region，还需选择对应的 **字段格式**。
2. 在 **IPv6数据库配置** 中选择数据来源（ip2region / GeoLite2 / IPDB）。若来源为 ip2region，同样需选择 **字段格式**。
3. 每个来源下方会显示该来源的数据状态，共三种：

   | 状态 | 含义 |
   | --- | --- |
   | ✓ 文件已就绪 | 该来源的数据库文件已上传到本地，正在使用你上传的文件。 |
   | ✓ 使用内置数据（可上传文件覆盖） | 未上传文件，但该来源有内置数据，可直接使用；上传文件后会覆盖内置数据。 |
   | ⚠ 文件不存在，请先上传 | 该来源既没有上传文件，也没有内置数据，需先到「上传数据库」标签页上传。 |

4. 配置卡片下方会展示 **当前状态**，包括数据来源、字段格式、文件大小和加载时间。若当前使用的是内置数据，数据来源后会带一个「内置」标签，且不显示「文件创建时间」（内置数据不对应本地文件）。
5. 点击 **保存配置**。仅当所选来源既无上传文件、也无内置数据时，系统才会弹窗提示，可选择「仍要保存」或先去上传文件。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 数据来源 | 该协议（IPv4/IPv6）使用的 IP 数据库来源：ip2region / GeoLite2 / IPDB。带「内置」标签表示当前用的是随程序发布的内置数据，而非你上传的文件。 |
| 字段格式 | 仅 ip2region 需要，用于匹配所用 xdb 文件的字段结构（见下表）。 |
| 文件大小 | 当前已加载数据库的大小（使用内置数据时即内置数据的大小）。 |
| 文件创建时间 | 数据库文件的创建时间；使用内置数据时不显示该项。 |
| 加载时间 | 数据库最近一次加载到内存的时间。 |
| 测试IP地址 | 用于测试查询的 IP 地址，支持 IPv4 和 IPv6。 |

### ip2region 字段格式说明

不同版本的 ip2region xdb 文件包含不同的字段，请根据所用 xdb 文件版本选择对应格式：

| 版本 | 字段结构 | 说明 |
| --- | --- | --- |
| 老版本格式 | 国家\|区域\|省份\|城市\|ISP | 系统内置的老版本格式，包含区域字段。 |
| 开源版格式 | 国家\|省份\|城市\|网络运营商 | ip2region 开源版标准格式，4 个字段。 |
| 满载版格式 | 大洲\|国家\|省份\|城市\|区县\|网络运营商\|其他 | 满载版格式，包含最完整的地理信息，7 个字段。 |
| 标准版格式 | 国家\|省份\|城市\|区县\|网络运营商\|其他 | 标准版格式，包含区县信息，6 个字段。 |
| 精简版格式 | 国家\|省份\|城市\|网络运营商\|其他 | 精简版格式，5 个字段。 |

## 常见问题

- **全新安装必须先上传数据库吗？** 不需要。默认的 IPv4（ip2region）和 IPv6（GeoLite2）都有内置数据，装好即可查询，页面上会显示「使用内置数据」。只有想换用更新或更详细的数据库时才需要上传。
- **上传后查询结果没变化？** 上传后需点击 **重新加载数据库**，使新文件生效。
- **保存配置时提示文件不存在？** 表示所选来源既没有上传文件，也没有内置数据（IPv6 的 ip2region、IPDB 属于这种情况），请先到「上传数据库」标签页上传后再保存。
- **支持的文件格式？** IPv4/IPv6 数据库支持 `.xdb`（ip2region）与 `.mmdb`（GeoLite2）；IPDB 支持 `.ipdb`。
- **想贡献其他数据库格式？** 可将数据库名称、格式说明及示例文件发送至 `samwafgo@gmail.com`，官方将评估并集成更多格式。
