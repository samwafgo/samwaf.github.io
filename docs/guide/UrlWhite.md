# URL白名单

## 功能简介

URL 白名单用于在 SamWaf 防护过程中放行指定的 URL。防护墙在进行检测时会忽略在白名单内的 URL，命中的请求将跳过相关防护检测直接放行。

适用场景：某些接口被规则误拦截（误报）时，可将其加入白名单，使其不再被防护规则拦截。

![URL白名单](/images/urlwhite.png)

## 操作步骤

1. 进入「URL白名单」页面。
2. 点击左上角的 **新建白名单URL** 按钮，打开新增弹窗。
3. 在弹窗中填写以下内容：
   - 选择 **网站**（需要应用该白名单的站点）。
   - 选择 **匹配方式**（等于 / 前缀匹配 / 后缀匹配 / 包含匹配）。
   - 填写 **URL**。
   - 填写 **备注**（可选）。
4. 点击 **确认** 保存。

> 列表上方可通过「网站」「URL」进行筛选搜索；勾选多条记录后可使用 **批量删除**，或使用 **清空所有** 一次性删除当前数据。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 网站 | 该白名单生效的站点，从已添加的网站列表中选择。 |
| 匹配方式 | URL 的匹配规则，可选：等于、前缀匹配、后缀匹配、包含匹配。 |
| URL | 需要放行的 URL，按所选匹配方式进行比对。 |
| 备注 | 自定义说明，便于后续管理（可选）。 |
| 创建时间 | 记录的创建时间（系统自动生成）。 |

### 匹配方式说明

- **等于**：请求 URL 与填写内容完全一致时命中。
- **前缀匹配**：请求 URL 以填写内容开头时命中。
- **后缀匹配**：请求 URL 以填写内容结尾时命中。
- **包含匹配**：请求 URL 中包含填写内容时命中。

## 常见问题

- **白名单和 URL 黑名单冲突怎么办？** 白名单用于放行、URL 黑名单用于阻止，二者用途相反。请根据实际需求只在一处配置，避免对同一 URL 同时设置相互冲突的规则。
- **加入白名单后是否完全不检测？** 命中白名单的 URL 会被放行并忽略相关防护检测，请谨慎为重要接口配置白名单。
