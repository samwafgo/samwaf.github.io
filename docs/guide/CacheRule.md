# 缓存规则

## 功能简介

缓存规则用于对匹配的请求进行缓存处理，对命中规则的请求直接返回缓存内容，从而降低后端压力、提升访问速度。不同的规则类型适用于不同场景。

## 前置条件

缓存规则需要先在对应网站上开启 **缓存** 功能：进入网站编辑页，将「缓存」设置为开启，并选择缓存位置（内存 / 文件 / 全部）后，缓存规则才会生效。

## 操作步骤

1. 点击左上角的 **新建**。
2. 在弹窗中选择 **网站** 并填写 **规则名称**。
3. 选择 **规则类型**（后缀匹配 / 指定目录 / 指定文件），并按照规则类型填写对应的 **规则内容**。
4. 选择 **参数处理** 方式（忽略参数 / 完整参数）。
5. 设置 **缓存时间**、**优先级** 和 **请求方法**，必要时填写 **备注**。
6. 点击 **确定** 保存。

如需修改或删除，可在列表操作列点击 **编辑** 或 **删除**。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 网站 | 缓存规则所属的网站。 |
| 规则名称 | 规则的名称，便于识别。 |
| 规则类型 | 选择缓存规则的类型，不同类型适用于不同场景。 |
| 规则内容 | 根据规则类型填写相应内容：后缀匹配填写文件后缀（如 `.jpg`），指定目录填写目录路径（如 `/images/`），指定文件填写文件路径（如 `/index.html`）。示例：`.jpg;.png;.gif` 或 `/images/` 或 `/index.html`。 |
| 参数处理 | 选择是否忽略 URL 参数。忽略参数时，带参数的 URL 将与不带参数的 URL 共用缓存。 |
| 缓存时间 | 缓存的有效时间，单位为秒。默认 3600 秒（1 小时）。 |
| 优先级 | 规则优先级，数字越大优先级越高。默认为 1。 |
| 请求方法 | 指定适用于哪些 HTTP 请求方法，多个方法用分号分隔。默认为 `GET`。示例：`GET` 或 `GET;HEAD`。 |
| 备注 | 自定义补充说明。 |

### 规则类型

| 类型 | 说明 |
| --- | --- |
| 后缀匹配 | 按文件后缀匹配，规则内容填写文件后缀（如 `.jpg`）。 |
| 指定目录 | 按目录匹配，规则内容填写目录路径（如 `/images/`）。 |
| 指定文件 | 按具体文件匹配，规则内容填写文件路径（如 `/index.html`）。 |

### 参数处理

| 选项 | 说明 |
| --- | --- |
| 忽略参数 | 带参数的 URL 与不带参数的 URL 共用同一份缓存。 |
| 完整参数 | 按完整 URL（含参数）区分缓存。 |
