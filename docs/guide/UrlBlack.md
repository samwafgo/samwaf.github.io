# URL限制访问

## 功能简介

URL 黑名单（URL 限制访问）用于阻止访问指定的 URL。防护墙会阻止访问在限制访问内的 URL，命中的请求将被拦截。

适用场景：禁止访问后台管理路径、调试接口、临时下线的页面等敏感或不希望对外开放的 URL。

![URL限制访问](/images/urlblack.png)

## 操作步骤

1. 进入「URL限制访问」页面。
2. 点击左上角的 **新建黑名单URL** 按钮，打开新增弹窗。
3. 在弹窗中填写以下内容：
   - 选择 **网站**（需要应用该限制的站点）。
   - 选择 **匹配方式**（等于 / 前缀匹配 / 后缀匹配 / 包含匹配）。
   - 填写 **URL**。
   - 填写 **备注**（可选）。
4. 点击 **确认** 保存。

> 列表上方可通过「网站」「URL」进行筛选搜索；勾选多条记录后可使用 **批量删除**，或使用 **清空所有** 一次性删除当前数据。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 网站 | 该限制生效的站点，从已添加的网站列表中选择。 |
| 匹配方式 | URL 的匹配规则，可选：等于、前缀匹配、后缀匹配、包含匹配。 |
| URL | 需要阻止访问的 URL，按所选匹配方式进行比对。 |
| 备注 | 自定义说明，便于后续管理（可选）。 |
| 创建时间 | 记录的创建时间（系统自动生成）。 |

### 匹配方式说明

- **等于**：请求 URL 与填写内容完全一致时命中并阻止。
- **前缀匹配**：请求 URL 以填写内容开头时命中并阻止。
- **后缀匹配**：请求 URL 以填写内容结尾时命中并阻止。
- **包含匹配**：请求 URL 中包含填写内容时命中并阻止。

## 常见问题

- **URL 黑名单和 URL 白名单的区别？** 黑名单用于阻止访问指定 URL，白名单用于放行并忽略防护检测。请勿对同一 URL 同时配置相互冲突的规则。
- **包含匹配是否会误伤其他路径？** 包含匹配的范围较广，凡是 URL 中包含填写内容的请求都会被阻止，请谨慎使用，必要时改用更精确的“等于”或“前缀匹配”。
