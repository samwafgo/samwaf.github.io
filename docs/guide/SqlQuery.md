# SQL 查询工具

## 功能简介

SQL 查询工具用于直接在 SamWaf 的本地数据库上执行查询语句，并查看数据库的表结构信息。它相当于命令行 `execsql` 的界面化版本，方便排查数据、核对统计或临时检索记录。

该工具面向**只读查询**场景：请输入 `SELECT` 查询语句（输入框占位提示即为「请输入 SELECT 查询语句」）。在数据库上直接执行语句存在风险，请谨慎使用，避免对核心数据造成影响。

页面分为两个标签页：**SQL 查询** 与 **数据库结构**。三类数据库可供选择：本地数据库（local）、日志数据库（log）、统计数据库（stats）。

## 操作步骤

### 一、执行 SQL 查询

1. 进入「SQL 查询工具」页面，停留在「SQL 查询」标签页。
2. 在 **选择数据库** 下拉框中选择目标库（本地 / 日志 / 统计）。
3. 在 **SQL 查询语句** 文本框输入 `SELECT` 语句。
4. 点击 **执行查询**，结果会以表格形式展示，并显示「共 N 条记录」。结果区支持分页浏览，空值显示为 `NULL`。

<!-- 图：查询结果表格 -->

### 二、查看数据库结构

1. 切换到「数据库结构」标签页。
2. 在下拉框选择目标数据库，点击 **刷新** 加载该库的所有用户表。
3. 页面顶部展示 **表数量** 与 **总行数** 汇总；表格列出每张表的表名、行数、字段数、索引数（默认按行数倒序，可点击列头排序）。
4. 对每张表可执行：
   - **查询 Top100**：自动填入 `SELECT * FROM 表名 LIMIT 100` 并切换到查询标签页执行。
   - **详情**：打开弹窗查看该表的字段信息（序号、字段名、类型、非空、默认值、主键）与索引信息（索引名、唯一、类型、包含字段）。

## 字段说明

### SQL 查询

| 字段 / 按钮 | 说明 |
|-------------|------|
| 选择数据库 | 查询目标库：本地数据库 / 日志数据库 / 统计数据库。 |
| SQL 查询语句 | 待执行的 SQL，输入 `SELECT` 查询语句。 |
| 执行查询 | 提交并执行查询，结果在下方表格展示。 |
| 查询结果 / 共 N 条记录 | 结果区标题与命中记录总数，空值显示为 `NULL`。 |

### 数据库结构

| 列 | 说明 |
|----|------|
| 表名 | 数据表名称。 |
| 行数 | 该表的记录行数（千位分隔显示）。 |
| 字段数 | 该表的列数量。 |
| 索引数 | 该表的索引数量。 |
| 操作 | 查询 Top100 / 详情。 |

表详情弹窗中的字段信息列：序号、字段名、类型、非空、默认值、主键；索引信息列：索引名、唯一、类型、包含字段。

## 常见问题

- **可以执行更新 / 删除语句吗？** 该工具面向只读查询设计，请输入 `SELECT` 语句。在数据库上直接执行语句有风险，请谨慎操作，避免破坏核心数据。
- **能查询哪些数据库？** 本地数据库、日志数据库、统计数据库三类。
- **查询结果有数量上限吗？** 结果以分页形式展示，便于浏览大批量记录；为避免一次返回过多数据，建议在 SQL 中使用 `LIMIT` 控制范围。
- **它和命令行有什么关系？** 它是命令行 `execsql` 的界面化等价工具，能力相近，便于在管理端直接查询。
