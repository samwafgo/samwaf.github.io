# URL隐私防护

## 功能简介

URL 隐私防护（数据脱敏处理）用于对指定 URL 返回的敏感数据进行脱敏处理。用户在访问指定的 URL 时，返回内容中的敏感数据会被脱敏，例如手机号会隐藏部分字符。

适用场景：列表或详情接口返回了手机号、身份证号等个人隐私信息，需要在不改动后端代码的情况下对外做脱敏展示。

![URL隐私防护(数据脱敏处理)](/images/ldp.png)

## 操作步骤

1. 进入「URL隐私防护」页面。
2. 点击左上角的 **新建隐私保护Url** 按钮，打开新增弹窗。
3. 在弹窗中填写以下内容：
   - 选择 **网站**（需要应用脱敏的站点）。
   - 选择 **匹配方式**（等于 / 前缀匹配 / 后缀匹配 / 包含匹配）。
   - 填写 **URL**。
   - 填写 **备注**（可选）。
4. 点击 **确认** 保存。

> 列表上方可通过「网站」「URL」进行筛选搜索；勾选多条记录后可使用 **批量删除**，或使用 **清空所有** 一次性删除全部隐私保护 URL 数据。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 网站 | 该脱敏规则生效的站点，从已添加的网站列表中选择。 |
| 匹配方式 | URL 的匹配规则，可选：等于、前缀匹配、后缀匹配、包含匹配。 |
| URL | 需要进行返回数据脱敏的 URL，按所选匹配方式进行比对。 |
| 备注 | 自定义说明，便于后续管理（可选）。 |
| 创建时间 | 记录的创建时间（系统自动生成）。 |

### 匹配方式说明

- **等于**：请求 URL 与填写内容完全一致时命中并执行脱敏。
- **前缀匹配**：请求 URL 以填写内容开头时命中并执行脱敏。
- **后缀匹配**：请求 URL 以填写内容结尾时命中并执行脱敏。
- **包含匹配**：请求 URL 中包含填写内容时命中并执行脱敏。

## 常见问题

- **脱敏会修改原始数据吗？** 脱敏只作用于返回给访客的响应内容，不会修改后端实际存储的数据。
- **支持哪些数据的脱敏？** 系统对返回内容中的敏感数据（如手机号）进行部分隐藏，具体以实际返回展示为准。
