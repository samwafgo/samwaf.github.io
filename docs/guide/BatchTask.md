# 批量任务

## 功能简介

批量任务用于自动执行一些批量化操作，每日凌晨 5 点依次执行，也支持手工触发。可将本地文件或远端 URL 中的数据批量导入系统，目前支持批量添加白名单 IP、黑名单 IP 和敏感词。

![批量任务界面](/images/batchtask_1.png)

## 操作步骤

### 1. 新建批量任务

点击左上角"新建"按钮，打开新建对话框。

![批量任务添加界面](/images/batchtask_2.png)

### 2. 填写任务信息

依次填写任务名称、网站、任务类型、额外配置、来源类型、来源值、执行方式、触发类型与备注。

- 选择不同的"任务类型"时，"额外配置"会自动填入对应的默认 JSON 模板（敏感词类型才需要额外配置，IP 类型为 `{}` 即可）。
- 来源类型为"本地路径"时，来源值填写服务器上的本地文件路径；为"远端 URL"时填写可访问的 http(s) 地址。

填写完成后点击"确认"保存。

### 3. 手工触发

在列表对应行点击"手工触发"，确认后即可立即执行该批量任务，无需等待每日定时执行。

### 4. 编辑与删除

- 点击"编辑"可修改已创建的批量任务。
- 点击"删除"并确认后即可移除该任务。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| 任务名称 | 自定义的任务标识名称（必填） |
| 网站 | 任务作用的网站，从下拉列表中选择（必填） |
| 任务类型 | 添加白名单 IP / 添加黑名单 IP / 添加敏感词（必填） |
| 额外配置 | JSON 格式的额外配置。敏感词类型支持 `check_direction`（检测方向：in 入站、out 出站、all 双向）与 `action`（检测后动作：deny 拒绝、replace 替换）；IP 类型无需配置（`{}`） |
| 来源类型 | 本地路径 / 远端 URL（必填） |
| 来源值 | 本地路径填写本地文件路径，远端 URL 填写 http(s) 地址（必填） |
| 执行方式 | 添加（在现有数据基础上追加）/ 覆盖（清空原有数据后写入）（必填） |
| 触发类型 | 定时任务（每日定时执行）/ 手动任务（仅手工触发）（必填） |
| 备注 | 可选的补充说明信息 |

## 常见问题

- **额外配置填写后无法保存？** 额外配置必须是有效的 JSON 格式，请检查括号、引号是否完整。
- **定时任务什么时候执行？** 触发类型为"定时任务"的批量任务每日凌晨 5 点依次执行；如需立即执行可使用"手工触发"。
