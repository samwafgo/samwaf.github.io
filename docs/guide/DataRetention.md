# 数据保留策略

## 功能简介

数据保留策略用于配置各类统计 / 日志数据的留存规则。SamWaf 会在**每天凌晨 04:30** 按照这里设置的策略自动清理历史数据，避免数据库无限增长占用磁盘。

页面顶部提示：策略由系统预置，**只可编辑「保留天数 / 保留行数」和「启用状态」**，不能新增或删除策略条目（目标表、归属库、时间字段等均为系统内置，只读）。

清理逻辑同时支持两个维度：

- **按时间清理**：超过「保留天数」的旧数据会被删除。
- **按行数清理**：超过「保留行数」上限的记录会被删除（一般保留最新的记录）。

两个值均可单独设置为 `0` 表示该维度不限制。

## 操作步骤

1. 进入「数据保留策略」页面，列表会展示所有预置的清理策略。
2. 点击右侧的 **编辑**，打开策略编辑弹窗。
3. 根据需要调整 **保留天数**、**保留行数**，并通过 **启用状态** 开关决定该策略是否生效。
4. 可在 **备注** 中补充说明。
5. 点击 **确定** 保存。系统会在下一次定时任务（每天 04:30）按新策略执行清理。

> 说明：弹窗中的「目标表」「归属库」为只读字段，仅展示该策略作用的数据库与数据表。「上次清理时间」「上次清理行数」为只读统计信息，反映最近一次自动清理的执行情况。

## 字段说明

| 字段 | 说明 |
|------|------|
| 目标表 | 该策略作用的数据库表名（系统预置，只读）。 |
| 归属库 | 该表所属的数据库类型（如统计库 stats 等，只读）。 |
| 保留天数 | 数据保留的天数，超过该天数的旧数据会被清理。`0` 表示不按天数限制。 |
| 保留行数 | 数据保留的最大行数，超过上限的记录会被清理。`0` 表示不按行数限制。 |
| 时间字段 | 用于判断数据新旧的时间列（系统预置，只读）。 |
| 启用状态 | 该清理策略是否生效。关闭后该表不会被自动清理。 |
| 上次清理时间 | 最近一次自动清理执行的时间，从未执行时显示「从未执行」（只读）。 |
| 上次清理行数 | 最近一次自动清理删除的记录条数（只读）。 |
| 备注 | 可选的说明信息。 |

## 常见问题

- **清理什么时候执行？** 每天凌晨 04:30 由定时任务自动执行，无需手动触发。
- **保留天数和保留行数同时设置会怎样？** 两个维度都会生效，任一条件命中的旧数据都会被清理；将某一维度设为 `0` 即可关闭该维度限制。
- **为什么不能新建 / 删除策略？** 策略条目由系统根据内置数据表预置，用户只能调整保留参数与启用状态，以保证清理逻辑安全可控。
