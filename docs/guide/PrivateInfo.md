# 密钥管理

## 功能简介

密钥管理用于集中保存各类敏感凭据（如 DNS 服务商的 API Key / Secret 等），并支持按「密钥分组」进行归类。所有密钥值都在本地加密保存，不会明文落库，也不会上传到任何远端服务。

最典型的使用场景是 **SSL 证书自动申请（DNS-01 验证方式）**：当你在「SSL 证书自动申请」中选择 DNS 验证时，需要填写对应云服务商（DNS 服务商）的 API 凭据，这些凭据就是通过「密钥分组 + 密钥管理」来组织和存放的。一个分组对应一套云服务商凭据，便于多账号、多云环境分类管理。

该页面合并了两部分内容：

- **密钥分组**：先建立分组，标明分组名称及归属云。
- **密钥管理**：在分组下维护具体的密钥键值对。

## 操作步骤

### 一、维护密钥分组

1. 进入「密钥分组」页面，点击 **新建密钥分组**。
2. 填写 **分组名称** 和 **分组归属云**。
3. 点击 **确定** 保存。列表中支持对已有分组进行 **编辑** 和 **删除**。

> 提示：在「SSL 证书自动申请」配置 DNS-01 验证时，会让你选择一个密钥分组；选定分组后即可在该分组下添加 / 编辑对应 DNS 服务商所需的密钥。

### 二、维护密钥

1. 进入「密钥管理」页面，点击 **新建**。
2. 填写 **密钥键**、**密钥值**，可选填 **备注**。
3. 点击 **确定** 保存。列表中支持对已有密钥进行 **编辑** 和 **删除**。

## 字段说明

### 密钥分组

| 字段 | 说明 |
|------|------|
| 分组名称 | 分组的自定义名称，用于在 SSL 申请等场景中选择。必填。 |
| 分组归属云 | 该分组归属的云服务商（DNS 服务商）标识，用于区分不同云平台的凭据。 |

### 密钥

| 字段 | 说明 |
|------|------|
| 密钥键 | 密钥的键名（如某 DNS 服务商要求的 API Key 字段名）。必填。 |
| 密钥值 | 密钥对应的值（如 API Key / Secret 的实际内容），本地加密保存。 |
| 备注 | 可选的说明信息，便于识别该密钥用途。 |

## 常见问题

- **密钥会明文保存吗？** 不会。密钥值在本地加密后落库，符合 SamWaf 私有化部署、数据不出本机的原则。
- **密钥分组和密钥是什么关系？** 分组用于归类（通常一个云服务商一组），密钥是分组下的具体键值。在做 DNS-01 证书申请时先选分组，再在分组下填具体凭据。
